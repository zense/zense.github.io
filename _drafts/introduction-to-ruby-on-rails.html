INTRODUCTION

Rails provides prewritten software functionality for creating web applications. It makes it easy to create common tools used in web development for database access, templating pages, authentication, etc. It was written using the Ruby programming language.

Rails follows 3 principles:

  1. DRY - Don't Repeat Yourself - By avoiding duplication, your code will be easier to maintain.
  2. Convention Over Configuration - Rails provides default ways of doing everything. This minimizes the amount of code we have to write to just our customizations. The Rails defaults also follow the best practices for web application development.
  3. MVC
    1. Model - Data
    2. View - What the user sees
    3. Controller - Responds to user events and helps the view communicate with the model.

By doing things this way, each time you change the View you don't have to change the model and vice versa.

Lets build something

Now that we have an fair outline about how requests are handled and fulfilled in a Rails application, let's use Rails to build a Twitter like application called Tooter.

To start off, lets create a new Rails project using:

$ rails new tooter -d mysql

The -d mysql part of the command specifies that we would like to use MySQL for the database. This creates a bunch of files in the folder:

$ ls
app  bin  config  config.ru  db  Gemfile  Gemfile.lock  lib  log  
public  Rakefile  README.md  test  tmp  vendor

Lets talk about these files and folders:

  1. app/ is where most of the code for your app will go:
    a. app/views/ : The interface for your app
    b. app/models/ : The data for your app
    c. app/controllers/ : Handles communication between the Model and View so that they can act independently.
  2. bin/ Contains files used to run your app
  3. config/ Configures your Routes, database
  4. config/routes.rb : Rails Router, when your app receives a URL, it calls the controller that provides the correct View
  5. config/database.yml : Database configuration file. Contains information about how to connect to the database. (More on it, later on)
  6. db/ Contains the database schema, which defines the tables and general organization of the database
  7. config.ru : Contains configuration rules for starting your app
  8. Gemfile / Gemfile.lock : Defines the Gems used by your app

Now, since we used MySQL for our app, we need to make a few tweaks to the config/database.yml file.

Open up your database.yml file and add your MySQL root password in the default -> password field. Also add a new field socket under default and set it to:

  1. /var/run/mysqld/mysqld.sock - for Ubuntu
  2. /var/mysql/mysql.sock - for OS X

Now run this command to create a database for your app:

$ bundle exec rake db:create

Now we can start our server using:

$ rails server
